### 阶段 1（本次必须完成）

## 三、语法与 API 处理策略（Edge 42）

### 3.1 必须替换/降级（若出现）

- 可选链：`obj?.a` → `obj && obj.a`
- 空值合并：`a ?? b` → `a != null ? a : b`
- `async/await`（若在目标环境不稳定）→ Promise 链式写法
- `AbortController`（兼容风险点）→ polyfill 或 `XMLHttpRequest + timeout` 兜底

### 3.2 可保留但建议审查

- `let/const`
- 箭头函数
- 模板字符串
- `fetch` / `Promise`

> 若保留，请在结果里明确说明“为什么 Edge 42 下可接受”。
